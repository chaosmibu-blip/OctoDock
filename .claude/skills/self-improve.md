# 自我改進檢查

## 什麼時候觸發

每次碰到以下情境時，自動執行這份檢查：

- 修正了一個 bug 或解決了一個問題
- 完成了一個新功能或修改了現有功能
- 發現了一個新的開發模式或最佳實踐
- 踩到了一個坑（無論有沒有解決）
- 收到用戶的反饋或糾正
- 架構、流程、介面發生了變化

## 檢查什麼

### 1. CLAUDE.md 需要更新嗎？

- 有沒有新的**開發原則**需要加入？（例如踩了某個坑，以後要避免）
- 有沒有**架構說明**過時了？（例如新增了模組、改了流程）
- 有沒有**文件索引**需要更新？（例如新增了文件）
- 有沒有**開發流程**改變了？（例如新增了步驟）

### 2. 現有的 skill 需要更新嗎？

- `adapter-quality-checklist.md` — 品質基準線有沒有新增的項目？
- `new-app-research.md` — 研究流程有沒有需要補充的步驟？
- 其他 skill — 有沒有過時的描述需要修正？

### 3. 需要建立新的 CLAUDE.md 規則嗎？

- 這次踩到的坑，能不能歸納成一條通用的開發原則？
- 有沒有新的架構模式、慣例、約定需要記錄下來？

### 4. 需要建立新的 skill 嗎？

- 這次的修正/開發過程中，有沒有產生了可以被重複使用的知識？
- 這個知識是不是只靠 CLAUDE.md 的一行規則不夠，需要更完整的 skill 文件？
- 新的 skill 能在未來類似情境中自動觸發嗎？

## 怎麼做

1. 如果需要更新或建立：直接做，不用問用戶
2. 如果不確定：提一句「我覺得需要在 CLAUDE.md 加一條規則 / 建立一個新 skill，你覺得呢？」
3. 更新後在 commit message 裡提到
